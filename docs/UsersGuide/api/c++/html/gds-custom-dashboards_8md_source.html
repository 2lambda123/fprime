1 # The GDS Dashboard
2 
3 The GDS Dashboard lets users create a customizable view for displaying data available to the GDS. Each of these views is
4 customizable for the project to display exactly what they need to see and nothing else. Users are able to choose view
5 components and modify them to suit project needs. Currently, there are 7 components to choose from:
6 
7 - `dashboard-box` groups components together visually
8 - `dashboard-row` helps organize components horizontally
9 - `command-input` sends commands to the embedded system
10 - `command-history` displays previously-sent commands
11 - `event-list` displays events the GDS receives
12 - `channel-table` displays telemetry data from the spacecraft
13 - `logging` displays logged messages stored as the GDS runs
14 
15 Through an XML specification, users can setup their dashboard and upload it to the F´ GDS to enable it in the system.
16 Once uploaded, the Dashboard should load the file and assemble the specified GUI as long as the XML is error free.
17 
18 A more complete reference is available here: [GDS Dashboard Reference](../dev/gds-dashboard-reference.md).

28 ## How to Use the Dashboard
29 
30 1. Start running the GDS GUI for your application by running `fprime-gds` in your deployment folder. Further
31  instructions can be found here: [GDS Introduction](./gds-introduction.md)
32 2. Navigate to the "Dashboard" tab using the navigation buttons at the top
33 3. Click "Browse" and select an XML configuration file containing your dashboard setup
34 4. The new Dashboard should render in the Dashboard tab.
35 
36 The browser will save your most recently-used Dashboard configuration and use it automatically when you restart the GUI.
37 At any point, you can upload a new configuration file to change your Dashboard.

39 ## Setting Up XML Configuration Files
40 
41 The dashboard is driven by loading an XML file which specifies the layout of this tab. In this, and the next few
42 sections the XML format will be discussed. At its most basic, the XML format is HTML with a few extra custom F´ tags
43 used to display our components. HTML may be used to help the user achieve the display they'd like, however; the entire
44 display can be configured from our custom components and thus no HTML knowledge is required.
45 
46 Here are some basic rules on how the layout will be displayed:
47 - Components will stack top to bottom using 100% width, unless components are inside a `dashboard-row`
48 - Components inside a `dashboard-row` will instead layout horizontally left to right.
49  - If multiple components are inside a `dashboard-row` the elements will split the width. e.g. Four components each
50  get 25% of the page width.
51 - `dashboard-box` and `dashboard-row` grow to contain all the components included and are limited to the maximum width
52 - Other components have a fixed height and variable width with scrollbars that appear when the component overflows
53 - Components will stay in the same row regardless of the page size and always maintain relative ordering
54 
55 The dashboard supports the following components shown below.
56 
57 | Component | Function | Available F´ Fields |
58 |----|----|----|
59 | <dashboard-row>...</dashboard-row> | Horizontal container for other components | |
60 | <dashboard-box>...</dashboard-box> | Box container for other components | title |
61 | <command-input/> | Command builder and transmission component | fields, filter-text, items-shown, compact |
62 | <command-history/> | List of commands sent as tracked by the GDS | fields, filter-text, items-shown, compact |
63 | <event-list/> | Ordered list of events received by the GDS | fields, filter-text, items-shown, compact |
64 | <channel-table/> | Latest channel readings as received by the GDS | fields, filter-text, items-shown, compact |

66 ## Customizing Components
67 
68 Individual components may be customized by passing in properties to them in the form of XML attributes. There are
69 several general properties available to many components. Some components allow for other properties that are shown in
70 the above table.
71 
72 | Property | Description | Type | Valid values |
73 |---|---|---|---|
74 | title | Title added as a label for box | string | any |
75 | color | Background color applied to box | string | [Any CSS color](https://www.w3.org/wiki/CSS/Properties/color/keywords) |
76 | border-color | Color of border around the box | string | [Any CSS color](https://www.w3.org/wiki/CSS/Properties/color/keywords) |
77 | fields | List of fields (columns) shown | list | any column field names |
78 | filter-text | Text to filter displayed items | string | any |
79 | items-shown | Names of items to restrict display | list | any |
80 | compact | Use minimalist display of component | keyword | |
81 
82 Two of the most basic properties are described below. These were chosen because they show an example of using a string
83 property and a space-separated list property. Keyword properties are specified without the `=...` portion and thus are
84 not described.
85 
86 ### Title Property
87 
88 The title property is used to add a title to other components. This helps label the display such that it is easier to
89 understand. The title property can be set by specifying the XML attribute as a string just as it is shown below.
90 
91 ```xml
92  <dashboard-box title="A Giant Blob of Data">
93  ...
94 ```
95 
96 ### Fields Property
97 
98 The fields property is used to restrict any of the table-driven components to minimize the number of columns that are
99 shown. Thus, users can show just the columns of interest and ignore the others associated with the table owned by the
100 component. This property is a list of values that are space-separated, and use single quotes (') to contain values that
101 included spaces. An example of both is shown below:
102 
103 ```xml
104  <channel-table fields="Description 'Channel Value' 'Channel Name'">
105  ```
106 
107 Each component type has different properties that can be used to customize it. All properties for components are
108 completely optional and omitted properties will use a default value. These properties are described in full below.

110 ## An Example Dashboard Configuration
111 
112 This section will provide an example dashboard configuration file. It shows how to compose various structures using the
113 structural components as well as examples of using the existing F´ display components. The dashboard we will explore
114 will be rendered into the following display.
115 
116 ![Example ](../media/example_dashboard.png)
117 
118 This view was rendered from the following XML. It is designed to be somewhat self-descriptive and should act as a good
119 starting point for building your own dashboards.
120 
121 ```xml
122 <dashboard-box title="The Big Green Example Box" color="green">
123  <dashboard-box title="This item is on top"/>
124  <dashboard-box title="But this item is below, by default"/>
125  <dashboard-row>
126  <dashboard-box title="Let's start a row instead"/>
127  <dashboard-box color="lightgreen">
128  <dashboard-box title="You can have boxes inside boxes..."/>
129  <dashboard-box title="...and they'll start..."/>
130  <dashboard-box title="...grouping by column again!"/>
131  </dashboard-box>
132  <dashboard-box>
133  <dashboard-box title="Since that's the default"/>
134  <dashboard-box title="And boxes don't need colors"/>
135  <dashboard-box color="coral" border-color="orangered">
136  <dashboard-box title="Or titles."/>
137  </dashboard-box>
138  </dashboard-box>
139  </dashboard-row>
140 </dashboard-box>
141 
142 <dashboard-row>
143  <dashboard-box title="A Clear Box"/>
144  <dashboard-box title="A Not-So-Clear Box" color="aqua"/>
145  <dashboard-box title="A Box with a Border" border-color="black"/>
146 </dashboard-row>
147 
148 <dashboard-box title="A More Practical Example" border-color="black">
149  <dashboard-box title="Control Panel" border-color="gainsboro">
150  <command-input/>
151  </dashboard-box>
152 
153  <dashboard-row>
154  <dashboard-box title="Warnings" color="lightyellow" border-color="yellow">
155  <event-list filter-text="WARNING"/>
156  </dashboard-box>
157  <dashboard-box title="Errors" color="salmon" border-color="red">
158  <event-list filter-text="FATAL"/>
159  </dashboard-box>
160  </dashboard-row>
161 </dashboard-box>
162 
163 <dashboard-box title="You can pass in space-separated lists to do things" border-color="black">
164  <dashboard-row>
165  <dashboard-box title="...like only showing certain columns">
166  <event-list fields="'Event Name' 'Event Description'"/>
167  </dashboard-box>
168  <dashboard-box title="...or picking out specific channels">
169  <channel-table items-shown="pingRcvr.PR_NumPings sendBuffComp.PacketsSent"/>
170  </dashboard-box>
171  </dashboard-row>
172 </dashboard-box>
173 ```

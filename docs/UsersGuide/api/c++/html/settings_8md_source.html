1 # `settings.ini`: Build Settings Configuration
2 
3 In many circumstances it is useful to set default values for the build as well as configure external
4 locations for F´ to use external code. The `settings.ini` file allows users to set various settings
5 to control the build.
6 
7 **Note:** settings.ini only adjusts the build when running through `fprime-util`. These settings can
8 be supplied directly to CMake using -D flags as described here: [CMake
9 Settings](../dev/cmake-settings.md), but it is not recommended to use CMake directly.

20 ## `settings.ini` Settings
21 
22 The `settings.ini` file is written in the INI format as interpreted by the default settings of the
23 Python `configparser` module. Should a user with to reference another key, use the `%(other key)s`
24 syntax. Full format description is available here:
25 
26 https://docs.python.org/3/library/configparser.html#supported-ini-file-structure
27 
28 This file is expected in the directory of the an F´ deployment and affects only the deployment it is
29 defined in. This directory is the same directory that contains the project's base `CMakeLists.txt`
30 (has a `project()` call).
31 
32 **WARNING:** The changes to the `settings.ini` file only take effect during the `fprim-util
33 generate` step, and thus builds **must** be regenerated after a change.
34 
35 ### `fprime` Section
36 
37 The `[fprime]` section defines settings for the F´ build.
38 
39 These settings include:
40 
41 - `project_root`: path to the root of the fprime project
42 - `framework_path`: Path to the F´ framework root
43 - `library_locations`: Paths to additional F´ libraries and components. Multiple paths can be
44  specified with the `:` separator. Ex: `../library1:../library2`
45 - `default_toolchain`: Default platform to build against. Defaults to `native`, or the host
46  computer platform.
47 - `environment_file`: An ini file that can be used to set environmental variables during the build
48  process.
49 - `ac_constants`: Path to autocode constants ini file.
50 - `config_directory`: Path to configuration header directory.
51 
52 ### Example `settings.ini`
53 
54 This `settings.ini` file comes from the [fprime-sphinx](https://github.com/fprime-community/fprime-sphinx)
55 deployment, which is a standard standalone deployment where the F´ framework and libraries are included
56 as git submodules at the top level.
57 
58 ```ini
59 [fprime]
60 project_root: ..
61 framework_path: ../fprime
62 library_locations: ../fprime-vxworks:../fprime-sphinx-drivers:../fprime-jplffs
63 default_toolchain: gr712-vxworks6
64 environment_file: ../fprime-vxworks/cmake/env/VxWorks-GR712.ini
65 ac_constants: ./Cfg/AcConstants.ini
66 config_directory: Cfg
67 ```

69 ### Example Environment Ini File
70 
71 The format for the environment file option is similar to `settings.ini`
72 Each key in the file will be set as environmental variable for the project build.
73 
74 ```ini
75 [environment]
76 WIND_BASE=/opt/WindRiver/vxworks-6.7
77 LINK_BIN_PRE=/opt/sparc-wrs-vxworks/bin/ccsparc
78 LINK_BIN_PRE_FLAGS=-r -nostdlib -Wl,-X
79 LINK_BIN_PRE_TO=-o
80 ```

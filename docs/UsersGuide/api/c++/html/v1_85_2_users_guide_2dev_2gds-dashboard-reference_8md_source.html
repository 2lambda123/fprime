1 # GDS Dashboard Component Reference
2 
3 The GDS dashboard is an XML specification for designing custom views into the GDS. This guide will walk through each
4 custom component that can be used in this specification. Properties are specified as XML attributes. This specification
5 in reality is HTML and thus HTML tags may be used. However, for best results, the user should stick to these components
6 where possible.

23 ## Available Components and Properties
24 
25 The dashboard is composed of components and properties set on those components. This section will give a quick overview
26 of the properties and the components that use them. Further explanation of the components is available below.
27 
28 The following is the list of all properties defined by the XML dashboard specification. These properties have types that
29 help the user enter them. These types are implicit to the property are quickly shown here:
30 
31 | Property Type | Description | Example |
32 |---|---|---
33 | string | text string | title="My fancy title" |
34 | list | Space separated list. Items may be quoted with (') | fields="Field1 'Field 2' 'Field 3'|
35 | keyword | Just a keyword, no value required | compact |
36 
37 The following are the properties available, the type of the property from the above table, and a description of valid
38 values.
39 
40 | Property | Description | Type | Valid values |
41 |---|---|---|---|
42 | title | Title added as a label for box | string | any |
43 | color | Background color applied to box | string | [Any CSS color](https://www.w3.org/wiki/CSS/Properties/color/keywords) |
44 | border-color | Color of border around the box | string | [Any CSS color](https://www.w3.org/wiki/CSS/Properties/color/keywords) |
45 | fields | List of fields (columns) shown | list | any column field names |
46 | filter-text | Text to filter displayed items | string | any |
47 | items-shown | Names of items to restrict display | list | any |
48 | compact | Use minimalist display of component | keyword | |
49 
50 Finally, here is a list of available components and which properties they allow.
51 
52 | Component | Function | Available F´ Fields |
53 |----|----|----|
54 | [<dashboard-row>...</dashboard-row>](#dashboard-row) | Horizontal container for other components | |
55 | [<dashboard-box>...</dashboard-box>](#dashboard-box) | Box container for other components | title |
56 | [<command-input/>](#command-input) | Command builder and transmission component | fields, filter-text, items-shown, compact |
57 | [<command-history/>](#command-history) | List of commands sent as tracked by the GDS | fields, filter-text, items-shown, compact |
58 | [<event-list/>](#event-list) | Ordered list of events received by the GDS | fields, filter-text, items-shown, compact |
59 | [<channel-table/>](#channel-table) | Latest channel readings as received by the GDS | fields, filter-text, items-shown, compact |

61 ## Dashboard Layout Components
62 
63 There are dashboard layout components, which help users organize their dashboard. Apart from constraining the layout,
64 these components provide no additional functionality. To understand these components and their properties, there are
65 two notes to help clarify:
66 
67 - Colors are specified as a [named CSS color](https://www.w3.org/wiki/CSS/Properties/color/keywords)
68 - All properties have defaults and are not required
69 
70 ### dashboard-box
71 
72 The `dashboard-box` acts like a container that other elements are put inside, helping to visually group items together.
73 This allows the user to specify arbitrary blocks of components that may optionally define a title, background color, and
74 border color. An example is shown below.
75 
76 ```xml
77 <dashboard-box title="A Box!" color="gray" border-color="black">
78  <!-- ...other content goes inside... -->
79  <command-input/>
80  ...
81 </dashboard-box>
82 ```
83 
84 `dashboard-box` supports the following properties:
85 - `title`: A text label to give the box, appearing at the top-left corner (defaults to nothing)
86 - `color`: The background color to give the box (defaults to transparent)
87 - `border-color`: The color to give the border of the box (defaults to transparent)
88 
89 ### dashboard-row
90 
91 The `dashboard-row` is a component for horizontal grouping. This can be used to force components or boxes to align
92 horizontally across the screen. Without the component, groupings would be arrayed vertically. The container itself is
93 invisible.
94 
95 ```xml
96 <dashboard-row>
97  <!-- ...other content goes inside... -->
98  <dashboard-box>...</dashboard-box>
99 </dashboard-row>
100 ```
101 
102 The `dashboard-row` allows no properties.

104 ## Functional Components
105 
106 The F´ dashboard allows for several functional components as well. These components allow some access to the GDS data,
107 and through the GDS, access to the embedded system. These components manipulate channels, events, and commands. File
108 handling components are currently not supported as they better suited for a full view.
109 
110 ### command-input
111 
112 `command-input` is used to build and send commands to the embedded system through the GDS. It consists of a command
113 drop down list, argument input boxes, a clear, and a send button. It is seen on the top of the `Commands` tab. The
114 input box is searchable. An example is shown below:
115 
116 ```xml
117 <command-input/>
118 ```
119 
120 `command-input` allows no properties.
121 
122 ### command-history
123 
124 `command-history` is a list of commands and arguments sent to the embedded system. It is important to note that this is
125 tracked through the GDS, and these commands may have not been successfully received by the embedded system. This list
126 can be filtered for command types, and reduced to a set of fields to be displayed. Use the [`event-list`](#event-list)
127 component with a filter for command severity to display commands as received by the embedded system. An example of the
128 `command-history` component is below:
129 
130 ```xml
131 <command-history
132  fields="'Command Time' 'Command Id' 'Command Mnemonic' 'Command Args'"
133  filter-text="cmdDisp"
134  items-shown="cmdDisp.CMD_NO_OP health.HLTH_CHNG_PING"
135  compact/>
136 ```
137 
138 `command-history` supports the following properties:
139 
140 - `fields`: A list of columns to *show*. The supplied columns must exactly match column headers (default: show all)
141 - `filter-text`: Initial filter text to filter items. Any items without string are hidden. (default: no filter)
142 - `items-shown`: A list of command mnemonics to display. Command mnemonics must exactly match. (default: show all)
143 - `compact`: Display using the compact version of this component.
144 
145 ### event-list
146 
147 `event-list` displays a set of events as received by the GDS. This allows the user to monitor for all commands, or
148 monitor for specific events. In addition, a filter may be used to look for certain severities. And example of the
149 `event-list` component is shown below:
150 
151 ```xml
152 <event-list
153  fields="'Event Time' 'Event Id' 'Event Name' 'Event Severity' 'Event Description'"
154  filter-text="WARNING_LO"
155  items-shown=""
156  compact/>
157 ```
158 
159 `event-list` supports the following properties:
160 
161 - `fields`: A list of columns to *show*. The supplied columns must exactly match column headers (default: show all)
162 - `filter-text`: Initial filter text to filter items. Any items without string are hidden. Use this for filtering by
163  severity or any arbitrary message text. (default: no filter)
164 - `items-shown`: A list of event names to display. Event names must exactly match. (default: show all)
165 - `compact`: Display using the compact version of this component.
166 
167 ### channel-table
168 
169 `channel-table` displays the most recent reading of a channel's value. It provides a near realtime look at the channels
170 in the embedded system as received by the GDS. Often users use the `items-shown` property to view specific channels of
171 interest and the `fields` property can be used to remove undesired information like channel id. An example is shown
172 below:
173 
174 ```xml
175 <channel-table
176  fields="'Last Sample Time' 'Channel Id' 'Channel Name' 'Channel Value'"
177  filter-text=""
178  items-shown="cmdDisp.CommandsDispatched"
179  compact/>
180 ```
181 
182 `channel-table` supports the following properties:
183 
184 - `fields`: A list of columns to *show*. The supplied columns must exactly match column headers (default: show all)
185 - `filter-text`: Initial filter text to filter items. Any items without string are hidden. (default: no filter)
186 - `items-shown`: A list of channel names to display. Channel names must exactly match. This is typically used to set
187  a specific subset of channels to monitor. (default: show all)
188 - `compact`: Display using the compact version of this component.

190 ## Misc Components
191 
192 This is a set of other components that are not typically used, but are nonetheless available.
193 
194 ### logging
195 
196 The logging component typically lives in the logging tab of the GDS. It may be used to show logs in a dashboard, but
197 this feature is rarely used.
198 
199 ```xml
200 <logging/>
201 ```
202 
203 `logging` supports no properties.

1 # GDS Custom Add-Ons
2 
3 The HTML GUI of the GDS is built around [Vue.js](https://vuejs.org/) components. Although fprime
4 defines a number of existing components that can be added to a custom dashboard, some users need to
5 build custom vue displays. If the display is purely HTML it can be done directly in the dashboard
6 but should custom code be required, a Vue.js component needs to be created. This can be done using
7 the add-on system.
8 
9 The add-on system allows developers to create add-ons, register them, and then they are available
10 for use within the custom dashboards. This gives access to all of Vue.js features for custom
11 dashboards.
12 
13 ## Implementation of an Add-On
14 
15 An addon for fprime just adds a custom JS file under the "flask/src" directory in the Gds system.
16 This JS file defines a Vue.js component in javascript with the HTML template and data top-level
17 items.
18 
19 
20 This file is typically placed in a folder named for the addon (e.g. `example`) and the JS file is
21 typically named `addon.js`. A quick example can be seen below.
22 
23 ```
24 Vue.component("example", {
25  template: "<div> template {{ key1 }} goes here</div>",
26  data: function() {"key1": "HTML"},
27 }
28 ```
29 
30 The fprime data structures (Channels, Events, etc.) can be tapped into using the datastore object
31 provided as a singleton from `datastore.js`. This singleton can be data-bound locally to receive
32 updates as new data arrives. Sample data binding is shown below.
33 
34 
35 ```
36 // Import the datastore
37 import {_datastore} from "../../js/datastore.js";
38 
39 Vue.component("example", {
40  // Add fprime Event count to our output template
41  template: "<div>Event count: {{ events.length }}</div>",
42  // Create local databinding from _datastore.events to events
43  data: function() {"events": _datastore.events},
44 }
45 ```
46 
47 Finally, most add-ons will provide a dashboard showing off the addon and usage therein. A very basic
48 dashboard can be seen below. The name of the component becomes the HTML tag available in the
49 dashboard. The dashboard file should be an XML file and is usually placed in the same directory as
50 the above JS file.
51 
52 ```
53 <dashboard-box title="Example" border-color="black">
54  <example></example>
55 </dashboard-box>
56 ```
57 
58 The last step is to import the addon in the `enabled.js` file found in the `addons/` directory. See below.
59 
60 ```
61 import "./example/addon.js"
62 ```

64 ## Vue.js Tips
65 
66 Although this documentation is not intended to give a full account of how to use [Vue.js](https://vuejs.org/), it will provide a few tips.
67 
68 1. Always define the HTML template with a single child element
69 2. `data` should be a function returning an object. This allows the component to be reused.
70 3. Choose a unique component name.
71 4. Vue.js usually runs in debug mode allowing for issues to appear in the JavaScript console.
72 
73 Available data from `_datastore` is described in the following table
74 
75 | Item | Type | Description |
76 |---|---|---|
77 | events | list | List of events received |
78 | channels | dict | Channel id to last received reading |
79 | upfiles | list | List of uplinked file objects |
80 | downfiles | list | List of downlinked file objects |
81 
82 A full example can be seen in the addons/image-display folder.

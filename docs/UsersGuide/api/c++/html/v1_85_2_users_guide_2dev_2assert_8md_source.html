1 # Asserts in F
2 
3 The F′ framework uses asserts to perform run-time checks for software
4 errors. They are a method for verifying conditions that should be true
5 unless there is a software or processor error. Assert.hpp defines macros
6 to declare an assertion in C++ code. CAssert.hpp defines macros to
7 declare an assertion in C code.
8 
9 The definition for the framework assert is found in Fw/Types/Assert.hpp.
10 The user calls the `FW_ASSERT(cond, arg1, ...)` macro with up to six arguments. The
11 arguments can consist of any basic types shown below.
12 
13  - I8 8-bit signed integer
14 
15  - U8 8-bit unsigned integer
16 
17  - I16 16-bit signed integer
18 
19  - U16 16-bit unsigned integer
20 
21  - I32 32-bit signed integer
22 
23  - U32 32-bit unsigned integer
24 
25  - F32 32-bit IEEE floating point number (float)
26 
27  - F64 64-bit floating point number (double)
28 
29  - I64 64-bit signed integer
30 
31  - U64 64-bit unsigned integer
32 
33  - bool C++ Boolean type
34 
35 These types are used in XML specifications. Note that not all types are
36 available on all processor architectures. The types that are available
37 is a configurable feature of the architecture and is typically set by
38 compiler arguments.
39 
40 The assert can be configured in the following ways:
41 
42  - FW\_ASSERT\_LEVEL Sets the level or reporting for the asserts.
43 
44  - FW\_NO\_ASSERT Shows the asserts turned off. The code to check
45  the condition is not compiled. Some developers prefer this once
46  the code has been tested to regain some processing performance.
47 
48  - FW\_FILEID\_ASSERT Identifies an integer value for the file
49  where the assert occurs (as opposed to \_\_FILE\_\_). It saves
50  code space since no file name is stored.
51 
52  - FW\_FILENAME\_ASSERT Identifies which file the assert occurred
53  in. (The \_\_FILE\_\_ macro is used.)
54 
55  - FW\_ASSERT\_TEXT\_SIZE Identifies the size of the buffer used to
56  store the text of the assert.
57 
58 ## AssertHook
59 
60 By default, when FW\_ASSERT is called the framework prints a message
61 (location of and arguments to the macro), and then calls the C assert()
62 function. The framework also provides a function that allows the
63 registration of a user-defined handler. The handler is
64 registerAssertHook and can be found in FwAssert.hpp. The assert hook is
65 called with a string representing the text of the assert. The user
66 implements a derived class that implements the reportAssert() pure
67 virtual method, and does whatever project specific logic is required.
68 
69 ### reportAssert
70 
71 When formatting an assert message use the default message line hash or
72 name, and argument approach.
73 
74 ### printAssert
75 
76 When outputting the assert message use the default Fw::Logger. Force the
77 Fw::Logger default to printf for systems with print f.

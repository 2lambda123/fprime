1 # Documentation of F´ Projects
2 
3 Most projects need to generate documentation om the software. F´ uses [Doxygen](https://www.doxygen.nl/index.html) to
4 generate project documentation and projects using F´ may use Doxygen to generate their project documents. Doxygen was
5 chosen because it can run automatically on F´ markdown and C++ documents to generate a web-compatible (HTML) version of
6 the documentation.
7 
8 The document will describe the basic usage of Doxygen for F´ projects and will also discuss some key settings that
9 projects may wish to change. The output of Doxygen will be a set of HTML files that allow inspection of the project
10 markdowns and C++ code. F´ Doxygen documentation may be viewed to see what the output will look like and is available at
11 [https://nasa.github.io/fprime/UsersGuide/api/c++/html/index.html](https://nasa.github.io/fprime/UsersGuide/api/c++/html/index.html).
12 
13 ## Basic Doxygen Usage
14 
15 Although a full understanding of Doxygen is outside of the scope of this document, we will attempt to discuss the basic
16 usage of doxygen as it pertains to F´. Doxygen requires two things a working directory and a configuration file. It
17 will recurse through the working directory generating documentation using the settings set the configuration.
18 
19 If the user wishes to include autocoded parts of F´ to the documentation, make sure to build your F´ project before
20 running Doxygen. Make sure to delete or exclude (see EXCLUDE settings below) any non-native builds before running.
21 
22 The working directory should be the root of the project that is to be documented. A configuration file can be generated
23 or the sample can be used/modified. To run Doxygen through the GUI, select the working directory and a configuration
24 file and then run generation. Alternatively the `doxygen` command line utility can be run. Change to the working
25 directory and supply the path to the configuration file. More detail can be found on the
26  [Doxygen](https://www.doxygen.nl/index.html) website.
27 
28 A sample configuration file can be seen at
29 [https://github.com/nasa/fprime/blob/devel/docs/doxygen/Doxyfile](https://github.com/nasa/fprime/blob/devel/docs/doxygen/Doxyfile).
30 This file can be used to run documentation of a project. Note: the output directory inside F´ will need to be changed
31 basic configuration settings are described below.
32 
33 Once generated, load `index.html` in the output directory using a web browser to view the documentation.

35 ## Basic Doxygen Configuration
36 
37 There are certain settings that users should change in the above configuration sample, and likely these settings should
38 be changed when starting from scratch.
39 
40 **Note:** The sample file and templated Doxygen configuration give a detailed explanation of each Doxygen setting.
41 
42 ### Key Settings
43 
44 These settings are essential to use Doxygen to generate documentation.
45 
46 | Setting | Description | Example |
47 |---|---|---|
48 | OUTPUT_DIRECTORY | Directory relative to working directory to write out HTML. | ./docs |
49 | STRIP_FROM_PATH | Paths to strip out of documentation. Add your project's build cache to this. | ./MyProject/build-fprime-automatic-native/ |
50 | STRIP_FROM_INC_PATH | Paths to strip from autocoder include files. Add your project's build cache to this. | ./MyProject/build-fprime-automatic-native/ |
51 | USE_MDFILE_AS_MAINPAGE | Set a MD file to be the landing page documentation | MyProject/README.md |
52 | MARKDOWN_SUPPORT | Pull in markdown files | YES |
53 | RECURSIVE | Run doxygen recursively | YES |
54 | EXCLUDE | Paths to exclude. Used to not generate fprime Doxygen inside project documentation. | ./fprime |
55 | EXCLUDE_PATTERNS | Wildcard patterns to not include | MyProject/test* |
56 
57 ### Project Descriptions
58 
59 There are several settings that describe the basic project these are described in the table below. A project should
60 change these settings.
61 
62 | Setting | Description |
63 |---|---|
64 | PROJECT_NAME | Name of project displayed prominently in the documentation |
65 | PROJECT_NUMBER | Project version number |
66 | PROJECT_BRIEF | Brief description of project |
67 | PROJECT_LOGO | 200x50 or smaller logo for project. Leave blank for no logo |

69 ## GitHub Pages Notes
70 
71 There is a small problem with the Doxygen files generated and hosting these pages on GitHub pages. This is because
72 Doxygen converts the leading / of markdown paths to an _ which then is ignored by GitHub pages autogenerator Jekyll.
73 
74 Possible fixes:
75 
76 1. Projects can set no-jekyll on their github pages. This will prevent converting MD to web pages, but the HTML output
77 of Doxygen will be hosted as-is. This is a good choice for projects that just want to host Doxygen output.
78 
79 2. Rename underscore files. This is necessary when Jekyll needs to preprocess MD files. A sample renaming script can
80 be found next to the Doxygen sample configuration. Mileage may vary.

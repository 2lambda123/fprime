6 This installation guide is specifically designed to enable individuals and researchers to get up and
7 running with F´ quickly. This guide is not intended for large teams with specific content
8 management (CM) requirements.
9 
10 **Note:** see the troubleshooting section at the bottom for help resolving common issues found during F´ installs.
11 
12 ## Requirements
13 
14 F´ depends on several items before the user should attempt to install it. These requirements are
15 listed below and should be on any system the user wishes to use. Should these items not be
16 available on the user's Operating System, then a Virtual Machine should be used. One option is
17 VirtualBox [https://www.virtualbox.org/](https://www.virtualbox.org/).
18 
19 Requirements:
20 
21 1. Linux or Mac OS X operating system (or Windows Subsystem for Linux on Windows)
22 2. CMake 3.5 or newer [https://cmake.org/download/](https://cmake.org/download/). CLI tool must be available on the system path.
23 3. Bash or Bash compatible shell
24 4. CLang or GCC compiler
25 5. Python 3.6+ and PIP [https://www.python.org/downloads/](https://www.python.org/downloads/)
26 6. Python Virtual Environment \* (pip install venv or pip install virtualenv)
27 
28 You can install them using this one-liner command :
29 
30 ```sudo apt-get update && sudo apt-get -y install cmake git python3-pip``` (for Debian based OS'es)
31 
32 OR
33 
34 ``` yum install cmake git python3-pip ``` (for RHEL OS)
35 
36 **Note:** it is possible to install and run F´ without a virtual environment, however; for
37 individuals and researchers, this is the recommended approach.
38 
39 ### Create and Activate a new Python Environment
40 
41 This will create a new virtual environment for F´ to be installed into. The following commands
42 will create a new virtual environment called `fprime-venv` and ensure that the virtual environment
43 is activated.
44 
45 **Python 3.6+:**
46 
47 ```
48 python3 -m venv ./fprime-venv
49 . ./fprime-venv/bin/activate
50 ```
51 
52 Any time the user wishes to use F´, this virtual environment should be activated. This should be
53 done in each new shell the user uses. This is all that is required to activate F´ for use.
54 
55 **Note:** file system for VirtualEnvironment should support symbolic links, or use the `--always-copy` flag to pip.
56 
57 **Activate the Virtual Environment**
58 
59 ```
60 . ./fprime-venv/bin/activate
61 ```
62 
63 ### Cloning the F´ Repository
64 
65 Installation of F´ requires cloning of the F´ core repository. This uses Git. This will bring
66 down the core framework and C++ files.
67 
68 **Clone F´**
69 
70 ```
71 git clone https://github.com/nasa/fprime
72 ```
73 
74 ### Installing F´ Python Requirements
75 
76 F´ python-support packages have certain requirements on various Python packages available on PYPI.
77 These requirements are broken out into two files one for the Autocoder tools and one for the GDS
78 tools package. This is to enable users to choose which tools they'd like to use.
79 
80 **Installing F´ Python Packages**
81 
82 ```
83 pip install --upgrade fprime-tools fprime-gds
84 ```

86 ## Checking Your F´ Installation
87 
88 The user may easily checkout that their F´ installation has succeeded by testing the following
89 commands. First, the user will test the build and Autocoder install. Next, the user will test the GDS
90 layer components. Should the user not have installed the given component, the commands won't work.
91 
92 **Testing F´ Autocoder Installation Via Building Ref Application**
93 
94 ```
95 cd Ref
96 fprime-util generate
97 fprime-util build --jobs "$(nproc || printf '%s\n' 1)"
98 ```
99 
100 **Testing F´ GDS Installation Via Running HTML GUI**
101 
102 ```
103 fprime-gds -g html -r <path to fprime checkout>/Ref/build-artifacts
104 ```
105 
106 **Note:** `Ref` should contain pre-built dictionaries and binaries for the user's system. This can
107 be achieved by running the Autocoder installation test (the user must have a working Autocoder
108 installation).

114 ## Troubleshooting
115 
116 This section will add some known hints to trouble-shooting with the installation of F´. This will hopefully help users
117 install things more effectively.
118 
119 ### Ubuntu, Debian, Python PIP, and Python Virtual Environments
120 
121 Ubuntu and possibly other Debian variants don't include the virtual environment nor pip packages in the default python 3
122 installation. To get a fully functional Python 3 installation, use these commands on Ubuntu:
123 
124 ```
125 sudo apt install python3 python3-pip python3-venv
126 ```
127 
128 Now you should be able to run the installation without trouble.
129 
130 ### Mac OS X and CMake Command Not Found
131 
132 If the user chooses to install CMake directly from the CMake site (not using homebrew nor Mac Ports), then the CMake command-line tools must be added to the user's PATH or default system libraries. The quickest command to do that is:
133 
134 ```
135 sudo "/Applications/CMake.app/Contents/bin/cmake-gui" --install
136 ```
137 
138 More information can be found here: https://stackoverflow.com/questions/30668601/installing-cmake-command-line-tools-on-a-mac
139 
140 ### System Python, Packages, and Python3
141 
142 Many operating systems offer python PIP packages through their package manager (apt, yum, etc). Most python projects
143 recommend avoiding those packages, but rather installing from PIP in a virtual environment. The reason for this is that the
144 version of the python package from the OS may not be the required version that the python project depends on. Thus, for
145 F´ we strongly recommend using a virtual environment and not system packages.
146 
147 If the user chooses to use system python as their python to run F´, they must ensure that "python" available on the path
148 points to a Python3 install, not the Python 2 installation that most OSes default to. Also, packages F´ uses when installing
149 may break the system Python, and prevent the OS from functioning. Use at your own risk.

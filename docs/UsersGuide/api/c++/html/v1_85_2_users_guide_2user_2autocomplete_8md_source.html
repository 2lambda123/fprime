1 # Autocomplete: Tab Completion for Command-Line Tools
2 
3 This guide will walk the user through installing command line tools's autocomplete feature. This works when being run on
4 many common shells including: bash, fish, and tcsh. Due to the support library we use, we are unable to support shells
5 outside of the shells supported by this library.
6 
7 While you can start using the GDS command line tools immediately after installing F´, turning on autocomplete will
8 require the few additional steps described herein. Using autocomplete is as simple as pressing the `[TAB]` and the
9 active command line options will fill-in. If more than one option is available, you can hit `[TAB]` twice to display
10 all possible choices and continue typing until there is only one matching choice left.

22 ## Installing Autocomplete
23 
24 Once F´ is installed the following commands can be used to setup autocomplete. Autocomplete operates by registering some
25 variables into the active shell. Thus, the user has two options to work with these tools: manual setup in each shell and
26 activating along with the virtual environment. Manually activating per-shell requires a manual step, but does not
27 change the Python managed virtual environment file. Activating along with the virtual environment is convenient, but
28 edits a file that is managed by Python and issues may arise across Python versions.
29 
30 **Note:** it is possible to add these commands to a `.bashrc` or `.cshrc` file but this is outside the scope of this
31 guide.
32 
33 ### Manual Activation (Run In Each New Shell)
34 
35 These Commands can be run directly in each shell to provide autocomplete. This setup will end when the shell process is
36 closed, so they should be run on each new shell that is opened.
37 
38 **Bash:** `eval "$(register-python-argcomplete fprime-cli)"`
39 
40 **zsh**:
41 1. Enable `bashcompinit` by running the following:
42  ```bash
43  autoload -U bashcompinit
44  bashcompinit
45  ```
46 2. Run `eval "$(register-python-argcomplete fprime-cli)"`
47 
48 **fish**: Run `register-python-argcomplete --shell fish fprime-cli | source`
49 
50 **tcsh**: Run ``eval `register-python-argcomplete --shell tcsh fprime-cli` ``
51 
52 ### Activating With Python Virtual Environment
53 
54 Since most users run F´ through a virtual environment, it would be easy to place the above commands into the virtual
55 environment activation script. In this way, every time the user activates the environment to work with F´, autocomplete
56 is also activated. However, making these changes change the virtual environment's activation scripts, which could have
57 unanticipated effects should the virtual environment file format change. This is unlikely and reinstalling F´ while
58 omitting this section would fix any problems caused by this.
59 
60 **zsh pre-setup**: Enable `bashcompinit` by running the following:
61 ```bash
62 autoload -U bashcompinit
63 bashcompinit
64 ```
65 
66 **Bash/zsh**: `register-python-argcomplete fprime-cli >> $VIRTUAL_ENV/bin/activate`
67 
68 **fish**: `register-python-argcomplete --shell fish fprime-cli >> $VIRTUAL_ENV/bin/activate.fish`
69 
70 **tcsh**: `register-python-argcomplete --shell tcsh fprime-cli >> $VIRTUAL_ENV/bin/activate.csh`
71 
72 **Note:** to take advantage of these changes immediately, reactivate the virtual environment at this point.

74 ## Autocomplete Tool Support
75 
76 Autocomplete is currently available with the `fprime-cli` tool. Support for `fprime-util` and `fprime-gds` is coming
77 soon. Currently, you can use tab completion in the GDS CLI tool to complete sub-command names, options, file paths, and
78 even the available commands on the F´ instance you're running.
79 
80 ## Further Information
81 
82 F´ has its CLI tools written in Python, and we've implemented tab completion through the
83 [argcomplete](https://github.com/kislyuk/argcomplete) library. If you have any troubleshooting concerns, want to extend
84 these tools, or need to explore more advanced settings, this documentation would be the starting point for
85 troubleshooting.

87 ## Conclusion
88 
89 This guide walked the user through several options to install command line autocompletion where it is supported. The
90 user is completely capable of running without this support but it may prove helpful to some users.

16 # Primitive Types
17 
18 F´ provides convenient type names for C/C++ primitive types. These are established to simplify the writing of F´ code
19 across multiple systems as well as give an exact definition for each variable used. These types are described in the
20 following table and are available to both the ground system, events, channels, and commands, and the software itself.
21 
22 | F´ Type | C/C++ Type | Description |
23 |---------|------------|-------------------------|
24 | BOOL | bool | C++ boolean |
25 | I8 | int8_t | signed 8-bit integer |
26 | I16 | int16_t | signed 16-bit integer |
27 | I32 | int32_t | signed 32-bit integer |
28 | U8 | uint8_t | unsigned 8-bit integer |
29 | U16 | uint16_t | unsigned 16-bit integer |
30 | U32 | uint32_t | unsigned 32-bit integer |
31 | F32 | float | 32-bit floating point |
32 | F64 | double | 64-bit floating point |
33 | NATIVE_INT_TYPE | int | architecture dependent integer |
34 | NATIVE_UINT_TYPE | unsigned int | architecture dependent unsigned integer |
35 | POINTER_CAST | | integer of sufficient size to store a pointer for the architecture |
36 
37 **Note:** C/C++ types come from `stdint.h` and `stdbool.h`. The last three types above are not of set size, but are
38 architecture dependent. Should a project's architecture not support all these types, see:
39 [Configuring F´: Architecture Supported Primitive Types](../dev/configuring-fprime.md#architecture-supported-primitive-types)

41 ## Polymorphic Type
42 
43 F´ defines a type for use by the user that can represent any of the primitive types using the same storage space. This
44 is similar to a C `union` defined with fields of each above type.
45 
46 ### Setting Polymorphic Values
47 
48 The PolyType object can have a value assigned to it via the constructor or the *equals* operator:
49 
50 ```
51 PolyType myInt(123)
52 PolyType myFloat;
53 myFloat = 123.03
54 ```
55 
56 ### Getting Polymorphic Values
57 
58 The value stored in the PolyType object can be retrieved two ways:
59 
60 1) Cast the object to the type of the value:
61 ```
62 U32 val = (U32)pt;
63 ```
64 6) Use the get() method:
65 ```
66 U32 val;
67 pt.get(val);
68 ```
69 
70 In both cases, if the type being retrieved does not match the stored
71 type, the code will assert.
72 
73 ### Checking Polymorphic Values
74 
75 The PolyType instance has isXXX functions for checking what type is being stored, where XXX is the name of the type.
76 
77 ```
78 PolyType p(123);
79 if (p.isU32()) {
80  ...
81 }
82 ```

84 ## Complex Types
85 
86 The F´ framework supports several types of auto-generated complex types that can be used in the system, including use
87 in software, command, event, and channel types, and even with the F´ ground system. These types are:
88 
89 1. Enums: defined enumeration of values
90 2. Arrays: fixed-length container of other types
91 3. Serializable: defined composition of other types akin to a C++ `struct`/`class`
92 
93 These can all be designed and used in the modeling of F´ and the C++ implementation is autogenerated.
94 
95 ### Enums
96 
97 Enums are a fix set of values that can be referred to by name, but are store and transmitted as an integer type. The
98 auto-generator produces a class to wrap the type providing convenience functions.
99 
100 ### Arrays
101 
102 Arrays are fixed-length containers of other types. They must be type-homogeneous but can store any other single type.
103 
104 ### Serializables
105 
106 Serializables are field-value compositions of other types. They can be type-heterogeneous and may contain any other type
107 as a value. The autocoder will generate a class with accessor methods for the fields. See the full specification for
108 the serializable XML: [F´ XML Specifications: Serializables](../dev/xml-specification.md#serializable)
109 
110 ### C++ Classes
111 
112 When interacting only in the software and not within the F´ design layer, nor with the ground system the user may use
113 arbitrary C++ `class`/`struct` types. If these types are to be passed through a port invocation as an argument, they
114 should be defined as subclasses of the `Fw::Serializable` and define `serialize` and `deserialize` methods to be called
115 to perform the serialization.

118 
119 F´ supports many different types, included autocoded complex types.

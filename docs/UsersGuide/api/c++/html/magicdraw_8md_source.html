1 # Magic Draw Support (Deprecated)
2 
3 MagicDraw was once used as a design program to augment F´ development. As of F´ 2.0, it is discontinued as it is
4 difficult to use and develop in. The F´ 2.0 series of improvements will being integration of F´´ which will replace
5 MagicDraw and hand-editing designs as the recommended pattern.
6 
7 Some users may wish to use magic draw and this document tries to capture some aspects of its use so such users are not
8 completely unable to get started. **Note:** further MagicDraw use is at the user's own risk and support may not be
9 available as the OpenSource team does not use nor support this work.
10 
11 ## MagicDraw Notes
12 
13 F´ has a plugin to MagicDraw that can be used to generate Ai.xml files to use with the autocoder. It is available in the
14 `Autocoders/MagicDrawCompPlugin` folder of F´. It can be installed by following the README file there, although support
15 past MagicDraw 18.5 is not guaranteed.
16 
17 F´ has some MagicDraw support for existing components, however; given the disuse of these features, not all components
18 or examples will work. Open up the `Ref/Top/REFApplication.mdxml` file to begin. If import errors arise, the user will
19 need to fix them, and that is outside the scope of this document.
20 
21 Should problems arise when looking for "ISF-MD-Profile.mdzip" it is now located here: `Autocoders/MagicDrawCompPlugin/model`.
22 
23 At the time of writing this document, Ai.xml files generated with the MagicDraw plugin work correctly with the Autocoder
24 and there are no plans in place to change this interoperability. Ongoing support for the plugin and supplied design
25 files have been discontinued.

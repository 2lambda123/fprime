1 # Standard Pipeline
2 
3 The Test API relies on the Standard Pipeline to interface with the GDS.
4 
5 standard.py:
6 
7 This file contains the necessary functions used to setup a standard pipeline that will pull data from the middle-ware
8 layer into the python system. This layer is designed to setup the base necessary to running Gds objects up-to the layer
9 of decoding. This enables users to generate add-in that function above the decoders, and run on a standard pipeline
10 below.

19 #### class fprime_gds.common.pipeline.standard.StandardPipeline()
20 Bases: `object`
21 
22 Class used to encapsulate all of the components of a standard pipeline. The life-cycle of this class follows the
23 basic steps:
24 
25 > 1. setup: creates objects needed to read from middle-ware layer and provide data up the stack
26 
27 > 1. register: consumers from this pipeline should register to receive decoder callbacks
28 
29 > 1. run: this pipeline should either take over the standard thread of execution, or be executed on another thread
30 
31 > 1. terminate: called to shutdown the pipeline
32 
33 This class provides for basic log files as a fallback for storing events as well. These logs are stored in a given
34 directory, which is created on the initialization of this class.

41 #### setup(config, dictionary, logging_prefix='/home/kevin', packet_spec=None)
42 Setup the standard pipeline for moving data from the middleware layer through the GDS layers using the standard
43 patterns. This allows just registering the consumers, and invoking ‘setup’ all other of the GDS support layer.
44 :param config: config object used when constructing the pipeline.
45 :param dictionary: dictionary path. Used to setup loading of dictionaries.
46 :param logging_prefix: logging prefix. Logs will be placed in a dated directory under this prefix
47 :param packet_spec: location of packetized telemetry XML specification.

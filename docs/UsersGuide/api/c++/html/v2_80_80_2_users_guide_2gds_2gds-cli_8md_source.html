5 features without the need for a graphical environment. This enables projects easier access to F´ testing and operations
6 in more environments.

25 ## What is `fprime-cli`?
26 
27 The `fprime-cli` tool provides a command-line interface for working with F´s Ground Data System, as an alternative to
28 using the browser-based GUI. While graphical interfaces are often convenient, a command-line interface is still handy to
29 have for situations like the following:
30 
31 - Working remotely on a headless server
32 - Analyzing data with existing command-line tools
33 - Automating certain tasks through scripts
34 - Pacifying hardcore command-line users in your office
35 
36 In particular, the CLI tools for the GDS let you do 3 essential operations from the comfort of your own shell:
37 
38 - Send commands to the embedded system through the GDS layer
39 - View events emitted by the embedded into the F´ GDS
40 - Receive system telemetry through the GDS
41 
42 Let's see how you can use these for yourself!
43 
44 ## Getting Started
45 
46 `fprime-cli` is automatically installed when installing F´. In order to use it, make sure that your virtual environment
47 has been activated. To ensure the tool is ready, run `fprime-cli -h` and you should see the help usage. If an error is
48 received, ensure F´ is properly installed via [the installation guide](../../INSTALL.md) and that your virtual
49 environment has been activated.
50 
51 `fprime-cli` connects to the `fprime-gds` ground system layer. Should the user wish to run `fprime-cli` make sure to
52 start an instance of the `fprime-gds` in headless mode `fprime-gds -g none`. More information can be found in the
53 [F´ GDS guide](./gds-introduction.md). If the GDS is running on a non-standard port, supply the `-p` and `-id` flags
54 on any command below to connect to it. Without these flags, the default address of `127.0.0.1:50050` will be used.

56 ### The Basics
57 
58 All of the GDS CLI commands are run through the `fprime-cli` tool; if you run that all by itself, you should see a
59 help message like this:
60 
61 ```
62 $ fprime-cli
63 usage: fprime-cli [-h] [-V] {channels,command-send,events} ...
64 
65 provides utilities for interacting with the F´ Ground Data System (GDS)
66 
67 positional arguments:
68  {channels,command-send,events}
69 
70 optional arguments:
71  -h, --help show this help message and exit
72  -V, --version show program's version number and exit
73 ```
74 
75 We can see there are 3 "positional arguments" we can provide here, corresponding to the 3 commands we can run:
76 
77 - `channels` lets us receive the values of open telemetry channels on the embedded system
78 - `command-send` lets us send commands to the embedded system
79 - `events` lets us receive events the embedded system is recording
80 
81 You can use each command by running `fprime-cli [COMMAND NAME]`, e.g. `fprime-cli events` or `fprime-cli channels`. To
82 learn more about each one, you can include the `-h` or `--help` flag after the command to print out the individualize
83  command help messages (e.g. `fprime-cli events --help`) seen below.
84 
85 ```bash
86 $ fprime-cli events --help
87 usage: fprime-cli events [-h] [-d DICTIONARY] [-ip IP] [-p PORT] [-t SECONDS]
88  [-l] [-i ID [ID ...]] [-c COMP [COMP ...]]
89  [-s STRING] [-j]
90 
91 print out new events that have occurred on the F Prime instance, sorted by
92 timestamp
93 
94 optional arguments:
95  -h, --help show this help message and exit
96  -d DICTIONARY, --dictionary DICTIONARY
97  path from the current working directory to the
98  "<project name>Dictionary.xml" file for the project
99  you're using the API with; if unused, tries to search
100  the current working directory for such a file
101  -ip IP, --ip-address IP
102  connect to the GDS server using the given IP or
103  hostname (default=127.0.0.1 (i.e. localhost))
104  -p PORT, --port PORT connect to the GDS server using the given port number
105  (default=50050)
106  -t SECONDS, --timeout SECONDS
107  wait at most SECONDS seconds for a single new message,
108  then exit (defaults to listening until the user exits
109  via CTRL+C, and logging all messages)
110  -l, --list list all possible event types the current F Prime
111  instance could produce, based on the events
112  dictionary, sorted by event type ID
113  -i ID [ID ...], --ids ID [ID ...]
114  only show events matching the given type ID(s) "ID";
115  can provide multiple IDs to show all given types
116  -c COMP [COMP ...], --components COMP [COMP ...]
117  only show events from the given component name "COMP";
118  can provide multiple components to show events from
119  all components given
120  -s STRING, --search STRING
121  only show events whose name or output string exactly
122  matches or contains the entire given string "STRING"
123  -j, --json return the JSON response of the API call, with events
124  filtered based on other flags provided
125 ```

127 ### Supplying F´ Dictionary
128 
129 As mentioned, these CLI commands let you interact with the GDS through events and commands and telemetry channels.
130 Through a *project F´ dictionary* the CLI can understand what commands, events and telemetry channels are available.
131 Every F´ project deployment will have a `*Dictionary.xml` file that's created when the project's deployment is built
132 In the `Ref` example project, it'll be the `Ref/Top/RefTopologyAppDictionary.xml` file. By reading this file, the CLI
133 tool knows what to look for when it's reading or sending data to the GDS. If it doesn't know where the dictionary is,
134 then the CLI tool can't do much beyond printing out help messages.
135 
136 When you run one of the CLI commands, the tool will automatically look for files ending in `Dictionary.xml` in your
137 current working directory and use the first one it can find. This behavior is similar to how `fprime-gds` searches for
138 dictionaries it uses to construct the browser GUI. If it can't find any dictionaries, it will print out an error message
139 like this:
140 
141 ```
142 fprime-cli: error: No valid project dictionary found
143 ```
144 
145 Running commands from your project folder (e.g. inside `fprime/Ref`) should work to find a dictionary, but if you want
146 to run commands for a different deployment or project, you can specify the dictionary file exactly using the
147 `--dictionary` option with the file's path. `fprime-cli command-send --dictionary Ref/Top/RefTopologyAppDictionary.xml`.

149 ### Using the Tools
150 
151 The `fprime-cli` commands can roughly be grouped into 2 categories:
152 
153 - Ones that receive data through the GDS (`channels`, `events`)
154 - Ones that send data through the GDS (`command-send`)
155 
156 #### Receiving Events and Channels
157 
158 The first category contains `channels` and `events`. There are two very similar commands to respectively listen for any
159 telemetry channels or logged events coming from the embedded system through the GDS. These commands print out all the
160 data they receive to the console until the user stops them with `CTRL+C`. Since this can be quite a lot of data,
161 the user will often want to limit the data to items the user specifically cares about. There are several options to
162 filter which items are displayed:
163 
164 - `-i`/`--ids` only prints data matching specified numeric IDs. e.g. `fprime-cli events --ids 1 2 3` will display events
165  with IDs: 1, 2, or 3
166 - `-c`/`--components` only prints items generated by the specified component instance.
167  e.g `fprime-cli channels --components pingRcvr` will only display data sent from the component named `pingRcvr`
168 - `s`/`--search` only prints strings containing the (case-sensitive) search term you provide. e.g. `fprime-cli events
169  --search WARNING_LO` will only display `WARNING_LO` events
170  - Note that unlike the other two options, `--search` will only take a *single* search string
171 
172 These filters can be combined together to further narrow your search (e.g. `fprime-cli channels -c pingRcvr
173 -s WARNING_LO` will only show `WARNING_LO` events from `pingRcvr`).
174 
175 If you want to view a full list of all the events or channels available, you can use the `-l`/`--list` flag on a command
176 to list everything in the dictionary (e.g. `fprime-cli events -l` will list all known events for the project) and if
177 that's too much information, the same filtering options we just talked about work on `--list`'s output, too!
178 
179 If you ever need the full programmatic guts of a response, you can also pass in the `-j/--json` flag to print out the
180 full data objects being received in JSON format. While this will include some redundant information, it's guaranteed to
181 show you all the information available and allow interaction with other tools.
182 
183 #### Sending Commands
184 
185 In the second category is the `command-send` command, which can used to send commands through the GDS to the embedded
186 system. You can send commands by giving the *full name* of the command. These are typically formed by joining the
187 component instance name and command name with a '.' (e.g. `health.HLTH_CHNG_PING`). When the command needs arguments,
188 these can be passed in as a space-separated list using the `-args`/`--arguments` option. To send a command that does not
189 need arguments like `cmdDisp.CMD_NO_OP`, you would just run the following:
190 
191 ```
192 $ fprime-cli command-send cmdDisp.CMD_NO_OP
193 ```
194 
195 To send one that *does* need arguments like `health.HLTH_CHNG_PING`, the following would be run passing in all three
196 needed arguments.
197 
198 ```
199 $ fprime-cli command-send health.HLTH_CHNG_PING --arguments eventLogger 50 20
200 ```
201 
202 
203 **Note:** If you need to pass in a string with spaces as an argument, just enclose it in quotes `"like this"`.
204 
205 Similarly to the other CLI tools, you can run `fprime-cli command-send -l` to get a list of the available commands and
206 the arguments they take. The same filter options are available to search for particular commands. If you have tab
207 completion enabled, you can also double-tap tab to view a list of potential commands based on what you've typed so far.
